Push data ( our data )Push: Pushing in Git refers to the action of uploading your local commits and
changes to a remote repository. It allows you to share your work with others and
make it accessible to them. By executing the git push command, you send your
local commits to a specified branch in the remote repository, updating its history.
60
61
Pull Request (parent library)Pull: Pulling in Git is the process of retrieving and incorporating the latest changes from a remote repository into your local repository. It combines two actions: fetching the changes from the remote repository and automatically merging them into your current branch. The git pull command is used to perform this operation and ensure that your local repository is synchronized with the latest changes made by others.
62
63
In summary, pushing uploads your local changes to a remote repository, while pulling retrieves and merges the latest changes from a remote repository into your local repository.
